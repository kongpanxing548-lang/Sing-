## 一、问题背景

### 遇到的问题
- 早报任务无法自动推送到飞书群聊
- 提示权限不足：`need_user_authorization`
- 无法创建飞书文档
- 无法发送群消息

### 根本原因
飞书开放平台对机器人权限有严格的分类管理，需要在**应用权限**和**用户权限**两个维度分别配置，并完成用户授权流程。

---

## 二、权限分类说明

### 2.1 应用权限（Application Permission）
机器人以**应用身份**调用API所需的权限，不需要用户授权即可使用。

### 2.2 用户权限（User Access Token）
机器人以**用户身份**调用API所需的权限，必须用户授权后才能使用。

### 2.3 权限区别
| 维度 | 应用权限 | 用户权限 |
|------|----------|----------|
| 位置 | 权限管理 → 应用权限 | 权限管理 → 用户权限 |
| 授权方式 | 无需用户授权 | 必须用户点击授权 |
| 使用场景 | 机器人自身操作 | 操作用户的文档/消息 |

## 三、必需权限清单

### 3.1 应用权限（必须添加）

```
application:application:self_manage    # 应用自管理
im:message:send_as_bot                 # 以机器人身份发消息
im:chat:readonly                       # 读取群聊信息
drive:file:write                       # 写入云空间
cardkit:card:write                     # 发送卡片消息
```

### 3.2 用户权限（必须添加）

#### 文档相关
```
docx:document:create                   # 创建云文档
docx:document:readonly                 # 读取云文档
docx:document:write                    # 编辑云文档
docs:document.media:upload             # 上传文档媒体
```

#### Wiki相关
```
wiki:node:create                       # 创建Wiki节点
wiki:node:read                         # 读取Wiki节点
board:whiteboard:node:create           # 创建画板节点
```

#### 消息相关
```
im:message:send                        # 以用户身份发消息
im:chat:read                           # 读取群消息
```

#### 其他
```
offline_access                         # 离线访问（刷新token）
```

## 四、配置步骤

### 步骤1：登录飞书开放平台
访问 https://open.feishu.cn/ 并用管理员账号登录

### 步骤2：找到对应应用
- 应用ID：`cli_a93ae2fd7d38dbd3`（小运）
- 或 `cli_a937ce6b15a15cd5`（小璇）

### 步骤3：添加应用权限
1. 进入「权限管理」→「应用权限」标签
2. 搜索并添加：
   - `application:application:self_manage`
   - `im:message:send_as_bot`
   - `im:chat:readonly`
   - `drive:file:write`
   - `cardkit:card:write`

### 步骤4：添加用户权限
1. 切换到「用户权限」标签
2. 搜索并添加：
   - `docx:document:create`
   - `docx:document:readonly`
   - `docx:document:write`
   - `docs:document.media:upload`
   - `wiki:node:create`
   - `wiki:node:read`
   - `board:whiteboard:node:create`
   - `im:message:send`
   - `im:chat:read`
   - `offline_access`

### 步骤5：配置重定向URL
1. 进入「开发配置」→「安全设置」
2. 添加重定向URL：
   ```
   https://www.feishu.cn/
   ```

### 步骤6：发布新版本
1. 进入「版本管理与发布」
2. 点击「创建版本」
3. 填写版本号（如 1.0.1）
4. 填写更新说明
5. 点击「发布」

### 步骤7：用户授权
**方式一：网页授权**
1. 访问授权链接：
   ```
   https://open.feishu.cn/open-apis/authen/v1/index?app_id=cli_a93ae2fd7d38dbd3&redirect_uri=https://www.feishu.cn/
   ```
2. 登录飞书账号
3. 点击「允许」完成授权

**方式二：飞书APP授权**
1. 打开飞书APP
2. 进入与机器人的对话
3. 等待授权卡片出现
4. 点击「允许」完成授权

## 六、常见问题

### Q1：权限已添加但还是提示权限不足？
**解决**：必须创建**新版本**并**发布**后才能生效。

### Q2：用户授权后还是无法使用？
**解决**：
1. 检查权限是否添加到正确的分类（应用权限 vs 用户权限）
2. 检查重定向URL是否配置正确
3. 尝试取消授权后重新授权

### Q3：授权卡片没有弹出？
**解决**：
1. 在飞书「设置」→「隐私」→「授权管理」中取消旧授权
2. 重新触发授权流程

### Q4：如何查看已授权的应用？
**路径**：飞书APP → 「我」→ 「设置」→ 「隐私」→ 「授权管理」

## 七、关键要点总结

1. **权限分两类**：应用权限和用户权限都要配置
2. **发布新版本**：修改权限后必须发布新版本才能生效
3. **用户授权**：用户权限必须用户点击授权后才能使用
4. **重定向URL**：必须配置正确的回调地址
5. **授权管理**：可以在飞书隐私设置中查看和取消授权
