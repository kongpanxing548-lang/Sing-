# 小运-AI三重进化机制

> 让 AI 越用越懂你的自动化系统

---

## 概述

本系统实现 AI 的**持续自我进化**，通过三重机制让 AI 从对话中学习用户偏好，自动优化行为规则。

**核心目标**：不是模型变聪明，而是规则文件越来越懂你。

---

## 三重进化机制

### 第一重：官方 capability-evolver 技能

**来源**：ClawHub（35000+ 安装量）
**路径**：`/root/.openclaw/workspace/skills/capability-evolver/`

**工作原理**：
- 每 50 轮对话自动触发一次自我分析
- 分析对话历史，发现用户纠正/偏好模式
- 自动更新 CLAUDE.md 行为规则文件

**触发方式**：
- 自动：每 50 轮对话后
- 手动：发送 `/evolve` 命令

---

### 第二重：自研 ai-evolver 脚本

**路径**：`/root/.openclaw/workspace/scripts/ai-evolver.py`

**功能特性**：
- 分析最近 7 天对话记录
- 提取用户纠正（"不要..."、"别..."、"错了..."）
- 提取用户偏好（"我喜欢..."、"我习惯..."）
- 自动生成规则建议
- 更新 SOUL.md 行为规则
- 生成进化报告保存到 `diary/`

**触发方式**：
- 自动：每周日 10:00（上海时区）
- 手动：`python3 /root/.openclaw/workspace/scripts/ai-evolver.py`

**Cron 任务**：`ai-weekly-evolution`

---

### 第三重：手动即时记录

**记录位置**：`SOUL.md` → `## 从对话中学到的`

**记录时机**：
- 对话中发现新模式立即记录
- 用户明确反馈后即时更新
- 重大偏好后立即归档

**已记录内容**：
- 效率偏好："全部搞定" = 立即执行、无需确认
- 质量要求：容忍度极低、必须自检、「小运-」前缀
- 沟通风格：直接简短、行动胜于言语

```
AI 行为持续优化
```

## 使用指南

### 查看进化状态

```bash
# 查看最近进化报告
ls -lt diary/evolution-report-*.md | head -5

# 查看 SOUL.md 学到的内容
cat SOUL.md | grep -A 20 "从对话中学到的"

# 查看 capability-evolver 状态
/skills list | grep capability-evolver
```

### 手动触发进化

```bash
# 触发 capability-evolver
/evolve

# 触发 ai-evolver
python3 /root/.openclaw/workspace/scripts/ai-evolver.py
```

### 添加即时规则

直接编辑 `SOUL.md`，在 `## 从对话中学到的` 部分添加：

```markdown
### 新发现
- **偏好**: 用户喜欢...
- **禁用**: 用户讨厌...
```

## 最佳实践

1. **主动反馈** - 不满意时立即说"不要..."，进化机制会捕获
2. **定期回顾** - 每周查看进化报告，确认规则是否准确
3. **手动校准** - 自动规则有误时，手动修正 SOUL.md
4. **保持简洁** - 规则越多维护成本越高，只记关键偏好

## 技术细节

### ai-evolver.py 算法

```python
1. 读取最近 7 天 memory/*.md
2. 正则匹配用户纠正和偏好
3. 统计频率（>=2次生成规则）
4. 更新 SOUL.md 的"从对话中学到的"
5. 生成报告保存到 diary/
```

### Cron 配置

```json
{
  "name": "ai-weekly-evolution",
  "schedule": "0 10 * * 0",
  "tz": "Asia/Shanghai",
  "command": "python3 ai-evolver.py"
}
```
